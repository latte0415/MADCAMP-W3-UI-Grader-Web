# 서비스 설명
사용자가 입력한 URL을 기반으로, 해당 웹사이트의 UI/UX을 휴리스틱 평가하는 사이트
미래 지향적이지만 동시에 미니멀한 디자인의 웹사이트일 예정

# 사이트 구조
1. 메인 페이지
2. 마이페이지
3. 라이브러리
4. 상세 화면

# DB 스키마
## auth.users
- 기본 테이블 사용


# 페이지 상세 기획
## 탑 내비게이션 바
- 로그인/로그아웃 버튼: 구글/카카오톡/애플 로그인만 지원 예정. supabase sdk 사용 (최우측 상단)
- 라이브러리 (탭): 로그인되어 있을 경우만 visible. 클릭할 경우, 라이브러리 화면. (로그인/로그아웃 바로 왼쪽)
- 홈 (탭): 라이브러리 화면일 경우만 활성화. 클릭할 경우, 홈화면(최좌측 상단)

## 메인 페이지
### 목적
: 랜딩 페이지 역할. 적절한 디자인을 통하여, 브랜드 가치를 직관적으로 전달.

### 구성 요소
- url 입력창: 사용자가 ui 분석을 요청할 url 입력하는 입력창
- 분석하기 버튼
- 탑 내비게이션 바
- 로그인/로그아웃 버튼: 우측 상단에 위치

### 액션
- 로그인 버튼 클릭 -> 로그인 팝업 (구글/카카오톡 로그인만 지원 예정. supabase sdk 사용)
- url 입력되면, 분석하기 버튼 활성화
- 분석하기 버튼 클릭 -> 로그인 여부 확인 후, 안되어있으면 로그인 팝업. 그 이후, URL 유효한지 확인 후, 라이브러리 화면으로 이동

### API
- URL 유효한지 확인


## 라이브러리 화면
### 목적
: 현재까지 본인이 분석 요청한 웹사이트 분석 내용을 볼 수 있게 함.

### 구성 요소
- 웹사이트 UI 분석 리스트
    - 컴포넌트 구성: URL, 발견된 위험 요소 개수(위험도별), 실행 날짜

### 액션
- 컴포넌트 클릭하면 상세 화면

### API
- 해당 유저가 생성한 리스트 조회
